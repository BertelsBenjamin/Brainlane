A word or 167 about me

I am a classical pianist with a passion for web technology and code.

After graduating magna cum laude (Yep, very proud of it!) from the conservatory, I got back to the real world to work as a Branch Manager for a piano company.
The last three years I spent taking over Limburg by organizing concerts, leading company visits and giving lectures about contemporary and historical instruments.
Result is that I am a hands-on, problem-solving and independant communicator.

Outside of my job I am enrolled in a Front End Developer course since 2018 at the T2-Campus in Genk.
Having had a couple of different mentors, I learned to build websites like these.
Currently I am finishing this course by developing my graduation project. It will be a SaaS project made in Angular that will be used as a central application for the Service Department. Technicians, dispatchers and clients (mainly music schools) will be using the app to get an at all time accessible overview of their respective patrimonium.

Database & Hosting

By using (non-)relational databases and query languages like NoSQL and MySQL, it is possible for each type of user to write, read, update and delete data.

Why would you want to store data?

Think about all necessary data for running a web shop (orders, client login, client behaviour, ...)
You'll want to store that information in a database.

Of course you need servers to run your website 24/7. Guess what... We have them too!

Angular & Node.js

Angular is being developed and maintained by Google. You can rest assured that it's not going anywhere soon. Applications built with Angular and running on Node.js are considered to be robust, manageable and yet performing at the speed of light.

I love to use Angular as Javascript framework because of its clear syntax by using TypeScript. While using TypeScript, also web developers can use the power of Object Oriented Programming.
